# Test Cases for PetClinic

This folder contains comprehensive, optimized E2E test cases generated by examining the live application structure and workflows.

## Test Suite Structure

| Module | Functional Area | File | Description |
|--------|-----------------|------|-------------|
| **Owners** | Owner Creation | [TC001_Create_Owner.md](owner_management/TC001_Create_Owner.md) | Positive and Negative flows for adding new owners |
| **Owners** | Owner Search | [TC002_Find_Owner.md](owner_management/TC002_Find_Owner.md) | Searching for owners by name, partial match |
| **Pets** | Pet Management | [TC003_Add_Pet.md](pet_management/TC003_Add_Pet.md) | Adding pets to owners, type selection, validation |
| **Visits** | Visit Tracking | [TC004_Add_Visit.md](visit_management/TC004_Add_Visit.md) | Scheduling/recording visits for pets |
| **Vets** | Vet Listings | [TC005_View_Veterinarians.md](veterinarian_management/TC005_View_Veterinarians.md) | Viewing vet lists via UI and API (JSON/XML) |

## Key Workflows Discovered

1.  **Owner Onboarding**: `Find Owners` -> `Add Owner` -> Fill Form -> `Owner Information`
2.  **Pet Onboarding**: `Owner Information` -> `Add New Pet` -> Fill Form -> `Owner Information`
3.  **Visit Scheduling**: `Owner Information` -> `Add Visit` -> Fill Form -> `Owner Information` 
4.  **Reference Data**: `Veterinarians` (Read-only list, supports API export)

## API Endpoints Identified

-   `GET /owners/find.html` (Search Form)
-   `GET /owners` (List/Search Results)
-   `POST /owners/new` (Create Owner)
-   `POST /owners/{id}/pets/new` (Create Pet)
-   `POST /owners/{id}/pets/{petId}/visits/new` (Create Visit)
-   `GET /vets.json` (API Data)
-   `GET /vets.xml` (API Data)

## Execution Order
Suggested execution for full E2E flow:
1.  TC001-01 (Nav)
2.  TC001-02 (Create Owner) -> Output: `newOwnerId`
3.  TC003-01 (Add Pet to `newOwnerId`) -> Output: `newPetId`
4.  TC004-01 (Add Visit to `newPetId`)
5.  TC002-01 (Find the owner created in step 2)
